##鹏华基金净值抓取思路和踩坑
一开始的思路是找到和代码及净值相关的网址，找了之后发现，代码和有净值的网址不是同一个网址，这就稍微有点不便利。从官网首页最全的基金列表循环，最终获取最全的基金代码和Id，存在字典中。这里面涉及一个循环，就是根据基金代码的变化，获取对应网址的净值。在最初设想的时候，是把循环放在方法里面，然后思路就很混乱，因为如果在方法里进行循环，就要对每次循环获得的数据进行整理。我最常用的是列表和字典，但是一和循环处理在一起，就搞不清。之前出现过好多错误，要么就是把列表的初始定义放在循环内部，每次一循环都重新开始，没有在累加。还有就是列表的append用法，我用的也总是没那么轻松。今天出现的问题是，总是无法把循环后的数据传递出来。后面把循环移到后面，get_nav方法只是获取一个代码对应的净值相关。然后开始循环。循环获得的数据用字典储存，最后实现打印。
这期间还遇到某个代码在所求日期时没有对应的净值，当没有净值的时候，界面显示是什么样的，只要把对应界面的内容排除掉，剩下的就是能正确抓取到的数据。里面用了if、type函数，把错误的类型排除出去。如何再对净值采集进行优化，还有好多好多路要走。
