微电影搭建三

本来想多看几节视频跟着做下去，谁知道莫名其妙出了bug，怎么也找不到原因。（ps:找到原因了，因为多打了个逗号，真的狂悲狂喜心好累= =）访问前面写的代码，也出了问题。可见想要一帆风顺是真的难。为了避免像无头苍蝇乱转（其实我觉得这几天我就是），继续看视频写我自己的理解，顺便捋一捋这个逻辑。（以管理员登录为例）

1.安装Flask_wtf，然后在form下面导入基类（FlaskForm),这个类定义表单中的一组字段，每个字段都有对象表示，字段对象可附属一个或多个验证函数，验证用户提交的数据是否有效。字段和验证函数是直接从WTForms包中导入。
2.首先，定义LoginForm这个类。然后依次对需要呈现的字段进行定义。（账号、密码、登录）

3.然后到views下面，先从forms中导入LoginForm,然后找到对应的登录路由，在视图函数中，先对LoginForm进行实例化，form=LoginForm(),然后把form作为变量，传到render_template模板中，即form=form。这里一定要注意的是左边的form是参数名，是对应模板的占位符，右边的form是当前作用域中的变量，表示同名参数的值。然后在模板中，{{ form.account }},相当于HTML中的<input>元素。Flask-WTF要求应用配置一个密钥，提升应用的安全性。CSRF
回到views的登录视图中，用validate_on_submit()确认表单是否填写内容，如果填写了，用data获取填写的数据。app.route装饰器中需要加入methods参数，来告诉Flask,在URL映射中把这个视图函数注册为GET和POST请求的处理程序，如果没指定参数，默认为GET。
