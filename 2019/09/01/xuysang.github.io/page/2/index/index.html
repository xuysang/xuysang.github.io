好热呀

下午刚下完一场大雨，可是晚上没有风吹进来，热的不想动，为了要把这篇写掉，顺便晚上睡个好觉，开空调啦。
好快好快好快，一个月过去了。
回想这一个月，我觉得我对自己太好了，如果按照这种程度自学下去，别说半年，两年三年都不知道能学出什么。所以，我要调整。说来容易可是怎么做啊。更何况，我还想保住我的头发，养点生。总结一下这一个月，我所做的就是看视频，然后工作任务完成后看一点书，算法看了忘，背了忘，再看还是忘。然后也想不出要完成什么项目，视频学的数据库知识，也只是听老师讲一遍，没有自己动手实践。最最最可怕的是，上班我有时间学习，我却不知道从何下手，又是焦虑又无可奈何。主要还是我没有给自己制定学习计划，偏偏要学的内容又太多了，自己先乱了阵脚。
那么怎么优化我的学习呢。之前说的我对自己太好了，就是因为我不想看算法，我就停了下来，我有意识想躲避一些需要思考费脑力的事情，我就允许自己那样去做了。我如果真的想学好，就不能再这样啦。比如说算法，已经卡在排序这块好久了，应该试着刷题让自己深入理解，就算自己做不出来，背也要把别人优秀的答案背下来。so要开始刷算法题！！！数据库方面，光是看视频，自己不动手肯定不行。需要想一个实践项目，正好我学习需要一个方向，是web还是爬虫还是数据分析，这些都需要数据库，所以重点是找到项目练习。最后还有python的深度学习，这里面还涉及对计算机的底层实现，计算机操作系统啥的一些专业知识，我需要被科普和了解啊。
那怎么分配我学习时间，怎么才能在工作忙完后的空挡里有事情可做，心不慌。
七月明天就来了，七月会有一周的加班，这段时间能空出来的时间不大，我只有三周的较多可支配时间。我觉得工作日，我就不要看综艺电视剧和b站小视频了，平时如果真的认真学习了，需要更多的时间好好休息睡觉。都攒到周末一起看，好好放松，感觉应该也不错。周末可以专门空出一天玩，另外一天再梳理梳理学习情况，做适当的调整和补充学习。
那么从下周开始，我要看算法做算法题，找到合适的项目开始练习。算法适合在比较安静的环境下，那么下班后的两三个小时，比较适合。找项目，可以在上班期间多找找。如果上班期间还有空，又不知道应该做些什么，就写日志吧，写完回家发。

2019-06-14

与大晨星二三事

维护晨星评级给出的相关数据是我部门的一项业务。自从晨星页面改版，每页只能显示25条记录，配置和开发那边一直不解决这个问题，公司后来花了1000块买了采集晨星数据的业务。不久前，部门的一同事写了抓取代码成功抓取到了晨星数据，但是他写的代码提取不到最后几页的数据。这周，出于尝试的心态，我准备自己也写一个抓取的脚本，如果成功了，是不是意味着我的代码也值1000块了呢，奔着这个念头，我兴致勃勃地开动了。

要抓取快照和业务和风险这两字段下的数据。先来抓取快照。


到目前为止，快照有336页8000多条记录。通过点击下一页可以很直观的看到网址没有发生变化，这应该是通过ajax方法显示的。那么审查元素看一下，是否可以找到和页数有关的参数，直接构造一个新的url或者通过post传递参数，就可以找到随着页数变化的代码了。通过一通乱七八糟的操作后，我终于找到了相关参数。


猜测一下，_EVENTARGUMENT是页码，在拼接url连连失败后，我就把它作为参数传入，用post获取代码，满心期待返回的响应。

被当成黑客了？哇，我居然被认为是黑客耶！认真兴奋了一秒，我随即陷入了悲伤，是哪一步出现问题了呢？我明明把所有的参数都传进去了呀。_VIEWSTATE,_EVENTVALIDATION都是很长很长的混合字符，一开始我以为它们是固定的，仔细翻页比对后发现，它们是变化的。如果是变化着的，那我把不变的字符串传入进去，必然是错误的。找到这个原因后，我必须想办法找到这两个参数正确的值。百度了这两个参数后，我大概了解了这是个防御机制，避免一些恶意的访问，必须和参数的值一致才能跳转正确的源码。查阅了网上针对解决这两个参数的文章后，大概是这么一个思路，先通过get获取两个参数的值，再当做参数进行post。有了思路之后，我又按照这个思路写了代码。没有成功。经过反复的自我怀疑和参考网上的文章，我都做好通过selenium来抓取的准备了，最后不死心的又尝试了一下。定义两个参数为全局变量，因为随着页数的不断跳转，get和post请求一直在执行，变量一直在前后代码中变化。这一试，成功。

再来业绩和风险。我又陷入了自己的坑，绕着url和post参数，怎么都想不出办法。在同事的提点下，我post参数直接传入给定的字符串，终于找到了业绩与风险的源码。那么如何获取翻转页码后的源码呢？因为参数发生了变化，又回到了和快照一样的困境。好在当时及时被叫去开会，会上我依然在思考这个问题。我想到，既然可以先get获取参数再post传入参数，那也可以先post传入既定的参数，然后获取到对应的参数再进行post，实践了一下，发现果然行得通。至此，最困扰的解决了，其余的数据，只要不断调试就都可以抓取到。

2019-06-08

宁静的夏天

六月和五月是不一样的心情。

五月是焦虑的，但充满希望，六月是看似平和的，但却多了一丝妥协，向现实低头的无奈。我突然能理解前两年的自己，每天按部就班的工作，是很省心的，没有太多焦虑感，也不用逼自己非得怎样怎样学习提升。回到公司工作一周，那种熟悉感马上又产生了，舒适区又开始向我袭来笼罩着我，面试学习新的内容好像有了挡箭牌，没有之前那么紧迫了。

但是我很不喜欢，鸡血凉了，之前的想法好似一个笑话。可是我不知道该怎么学习，在公司，即使给我一个下午的时间支配，我也不能有效的学习，很奇怪。明明觉得时间应当是宝贵的，可把大把的时间呈现在我眼前的时候，我又没那么珍惜了。

最近听到不少悲伤的消息，村里的某某或是认识的某某阿姨，出了意外走了。一直以来，我经历的离别，都是生老病死让人无可奈何却也顺其自然的，突然出现意外离世的，在我印象中不多，所以给我带来了一定程度的冲击。就像有句歌词说的，意外和明天不知道哪一个先来临，随着他们的离世，总觉得有某种联系断了。其实认识的人很多，但真正能在日后碰上面的，往往是少数。但不管能不能见面，知道他们在世界上活得好好的，会有一种安心感，突然走了的人，联系就断了，是真的真的，再也见不到了。

这一周我写了几个小程序，一个是抓取某些基金公司的净值，时间可以自己给定；一个是自动识别资本市场网的验证码，获取ISIN编码，虽然验证码有时候会出现错误，但是只要提高程序的执行次数，应该可以提高成功率；一个是自动下载巨潮资讯的基金公告。但在写这些小程序的过程中，我有这样的意识，就是自己的代码是不太严谨的，因为我只是存粹实现了我想要实现的功能，但是对可能会有的bug以及优化措施，我犯了懒，不太愿意去想。另外在写程序的时候也暴露了，我还是没有及时的将经验归纳整理，每次都要重新去翻自己以前写的，还是不够熟悉和顺手。另一方面就是，我能想到的可以实现自动化的还是太少了，应该多想多练。我真的很想实现自动发送邮件的功能，下周一定要投入研究。

换了一个靠窗的位置，外面的风景还不错吧。

慢慢来比较快

希望一会儿要写的内容，不要被未来疯狂打脸。
经过一两个月的焦虑，现在相对平和下来了，可能是因为想明白了一些事情。我之所以焦虑，是因为突然对某个东西有了欲望，而自己的能力却不足以支撑。这里有一个词要划重点，是“突然”。不由的又要扯到大环境，现在可能真的是一个“速度”的时代，网络的快速发展将人的耐心不断的打磨，什么都要求快。看视频二倍速，似乎可以节约时间看更多的东西。时间变得越来越珍贵，可是也变得越来越零散，因为“快”不允许在一件事上面耗费太多时间，所以做的事情是多了，但是很可能只是在做那件事的当下获得了满足，过后就遗忘。这也是为什么我回想工作以来这两年，却不知道自己做了什么，白茫茫。另一方面，网络的普及，让成功的例子随处可见，让我以为，真的可以随随便便成功，以为自己是幸运儿，以为不用费多大力气或者短时间苦一下马上能获得成功。概括就是一个字，“飘”。这真的是挺恐怖的一个字，与它对立的是我们中华民族传统品德的“脚踏实地”。
“突然”是一个短时间的概念，它让人的情绪在短时间达到一个高度，从而带来了刺激、振奋的感觉。并且为此，可能会做出很多冲动的事情。这不是说突然产生的念头，突然产生的冲动是不好的，只是告诉自己，突然的背后，要花更长更长的时间，来为这个念头、这个冲动背书。做好准备了吗？
