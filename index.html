输入方式

今天来总结一下c语言中输入方式的选择。
首先，最常用的是用scanf(“%d”,&x)的方式来接收输入的单个字符或是数字。此时，如果需要连续输入，可以使用for循环（倘若不知道要输入多少，可以在之前设置需要输入数的个数）。这里还可以采用while(scanf(“%d”,&x)==1)的方式来连续输入信息。在Windows下，输入完毕先按Enter键，再按Ctrl+Z键，最后再按Enter键即可结束输入。

getchar()可以ch=getchar()来输入字符，一个个输入，while((c=getchar())!=EOF)同上，需要按Ctrl+Z键结束输入。

同时，数组是很有用的，用来储存数据。首先需要定义数组s，在scanf(“%s”,s)中，它会读入一个不含空格、TAB和回车符的字符串存入字符数组s，s前没有“&“符号。它遇到空白字符会停下来。

函数接口定义：
List Insert( List L, ElementType X );
其中List结构定义如下：
typedef struct Node PtrToNode;
struct Node {
ElementType Data; / 存储结点数据 /
PtrToNode Next; / 指向下一个结点的指针 /
};
typedef PtrToNode List; / 定义单链表类型 */
L是给定的带头结点的单链表，其结点存储的数据是递增有序的；函数Insert要将X插入L，并保持该序列的有序性，返回插入后的链表头指针。

      与大晨星二三事
    

  

      

    
    

      
        
维护晨星评级给出的相关数据是我部门的一项业务。自从晨星页面改版，每页只能显示25条记录，配置和开发那边一直不解决这个问题，公司后来花了1000块买了采集晨星数据的业务。不久前，部门的一同事写了抓取代码成功抓取到了晨星数据，但是他写的代码提取不到最后几页的数据。这周，出于尝试的心态，我准备自己也写一个抓取的脚本，如果成功了，是不是意味着我的代码也值1000块了呢，奔着这个念头，我兴致勃勃地开动了。


要抓取快照和业务和风险这两字段下的数据。先来抓取快照。



到目前为止，快照有336页8000多条记录。通过点击下一页可以很直观的看到网址没有发生变化，这应该是通过ajax方法显示的。那么审查元素看一下，是否可以找到和页数有关的参数，直接构造一个新的url或者通过post传递参数，就可以找到随着页数变化的代码了。通过一通乱七八糟的操作后，我终于找到了相关参数。


猜测一下，_EVENTARGUMENT是页码，在拼接url连连失败后，我就把它作为参数传入，用post获取代码，满心期待返回的响应。




被当成黑客了？哇，我居然被认为是黑客耶！认真兴奋了一秒，我随即陷入了悲伤，是哪一步出现问题了呢？我明明把所有的参数都传进去了呀。_VIEWSTATE,_EVENTVALIDATION都是很长很长的混合字符，一开始我以为它们是固定的，仔细翻页比对后发现，它们是变化的。如果是变化着的，那我把不变的字符串传入进去，必然是错误的。找到这个原因后，我必须想办法找到这两个参数正确的值。百度了这两个参数后，我大概了解了这是个防御机制，避免一些恶意的访问，必须和参数的值一致才能跳转正确的源码。查阅了网上针对解决这两个参数的文章后，大概是这么一个思路，先通过get获取两个参数的值，再当做参数进行post。有了思路之后，我又按照这个思路写了代码。没有成功。经过反复的自我怀疑和参考网上的文章，我都做好通过selenium来抓取的准备了，最后不死心的又尝试了一下。定义两个参数为全局变量，因为随着页数的不断跳转，get和post请求一直在执行，变量一直在前后代码中变化。这一试，成功。


再来业绩和风险。我又陷入了自己的坑，绕着url和post参数，怎么都想不出办法。在同事的提点下，我post参数直接传入给定的字符串，终于找到了业绩与风险的源码。那么如何获取翻转页码后的源码呢？因为参数发生了变化，又回到了和快照一样的困境。好在当时及时被叫去开会，会上我依然在思考这个问题。我想到，既然可以先get获取参数再post传入参数，那也可以先post传入既定的参数，然后获取到对应的参数再进行post，实践了一下，发现果然行得通。至此，最困扰的解决了，其余的数据，只要不断调试就都可以抓取到。

      栈与队列
    

  

      

    
    

      
        
栈和队列都是线性数据结构，并且栈和队列还蛮相似的，所以偷个懒，放在一起写。记得最开始学的时候，我也有疑惑，就是为什么要存在这种数据结构，这种不是相当于数组和链表的变形么，看了王争老师的解释：特定的数据结构是对特定场景的抽象。那么栈和队列的构造，就是为了一定程度上减轻使用数组和链表，因为前两者暴露了太多操作接口，使用时比较不可控，容易出错。
那什么是栈，什么是队列呢，不难理解，栈就是后进先出，一般都是想象一个凹槽，东西一个个放进去，最后放进去的是最先取出来的，大概是这么一个模型。队列就是先进先出，想象一个队伍，队尾不断有人进来，出去就是从队首先出去。所以最主要是要实现push和pop两个功能。
用数组和链表都可以实现栈。如果是用链表实现，压入和取出的操作都是在表头实现。






感觉用数组实现比较好理解。

		_setDimension: function () {
			var viewport   = F.getViewport(),
				steps      = 0,
				canShrink  = false,
				canExpand  = false,
				wrap       = F.wrap,

			wrap.add(skin).add(inner).width('auto').height('auto').removeClass('fancybox-tmp');

			wPadding = getScalar(skin.outerWidth(true)  - skin.width());
			hPadding = getScalar(skin.outerHeight(true) - skin.height());

			// Any space between content and viewport (margin, padding, border, title)
			wSpace = wMargin + wPadding;
			hSpace = hMargin + hPadding;

			origWidth  = isPercentage(width)  ? (viewport.w - wSpace) * getScalar(width)  / 100 : width;

							if (scrollOut) {
								body.css('overflow-x', 'hidden');

							origHeight = body.outerHeight(true);
						}

					} catch (e) {}
				}

			} else if (current.autoWidth || current.autoHeight) {
				inner.addClass( 'fancybox-tmp' );

				// Set width or height in case we need to calculate only one dimension
				if (!current.autoWidth) {
					inner.width( origWidth );
				}

				if (!current.autoHeight) {
					inner.height( origHeight );
				}

				if (current.autoWidth) {
					origWidth = inner.width();
				}

				if (current.autoHeight) {
					origHeight = inner.height();
				}

				inner.removeClass( 'fancybox-tmp' );
			}

			height = getScalar( origHeight );

			ratio  = origWidth / origHeight;

			minWidth  = getScalar(isPercentage(minWidth) ? getScalar(minWidth, 'w') - wSpace : minWidth);
			maxWidth  = getScalar(isPercentage(maxWidth) ? getScalar(maxWidth, 'w') - wSpace : maxWidth);

			minHeight = getScalar(isPercentage(minHeight) ? getScalar(minHeight, 'h') - hSpace : minHeight);
			maxHeight = getScalar(isPercentage(maxHeight) ? getScalar(maxHeight, 'h') - hSpace : maxHeight);

			// These will be used to determine if wrap can fit in the viewport
			origMaxWidth  = maxWidth;
			origMaxHeight = maxHeight;

			if (current.fitToView) {
				maxWidth  = Math.min(viewport.w - wSpace, maxWidth);
				maxHeight = Math.min(viewport.h - hSpace, maxHeight);
			}

			maxWidth_  = viewport.w - wMargin;
			maxHeight_ = viewport.h - hMargin;

			if (current.aspectRatio) {
				if (width > maxWidth) {
					width  = maxWidth;
					height = getScalar(width / ratio);
				}

				if (height > maxHeight) {
					height = maxHeight;
					width  = getScalar(height * ratio);
				}

				if (width < minWidth) {
					width  = minWidth;
					height = getScalar(width / ratio);
				}

				if (height < minHeight) {
					height = minHeight;
					width  = getScalar(height * ratio);
				}

				if (current.autoHeight && current.type !== 'iframe') {
					inner.width( width );

					height = inner.height();
				}

				height = Math.max(minHeight, Math.min(height, maxHeight));
			}

				inner.width( width ).height( height );

				// Real wrap dimensions
				width_  = wrap.width();
				height_ = wrap.height();

				if (current.aspectRatio) {
					while ((width_ > maxWidth_ || height_ > maxHeight_) && width > minWidth && height > minHeight) {

							height = getScalar(width / ratio);

							height = getScalar(width / ratio);

					height = Math.max(minHeight, Math.min(height, height - (height_ - maxHeight_)));
				}
			}

			if (scrollOut && scrolling === 'auto' && height < origHeight && (width + wPadding + scrollOut) < maxWidth_) {
				width += scrollOut;
			}

			inner.width( width ).height( height );

			wrap.width( width + wPadding );

			width_  = wrap.width();
			height_ = wrap.height();

			canShrink = (width_ > maxWidth_ || height_ > maxHeight_) && width > minWidth && height > minHeight;
			canExpand = current.aspectRatio ? (width < origMaxWidth && height < origMaxHeight && width < origWidth && height < origHeight) : ((width < origMaxWidth || height < origMaxHeight) && (width < origWidth || height < origHeight));

					height	: getValue( height_ )

				origHeight : origHeight,

				wPadding   : wPadding,
				hPadding   : hPadding,
				wrapSpace  : height_ - skin.outerHeight(true),
				skinSpace  : skin.height() - height
			});

			if (!iframe && current.autoHeight && height > minHeight && height < maxHeight && !canExpand) {
				inner.height('auto');
			}
		},
