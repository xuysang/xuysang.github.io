难

想要做好一件事，不那么简单。从小听故事，很佩服那些能快速学习而且学得好的人，不由的也希望自己也能做到。但是好像总使不上力，就像跑步，老师们总在说爆发力，可是我始终也感知不到我的爆发力，我只觉得自己尽了力，但是这个度到底怎么衡量，我也不知道。似乎我做什么事情，激情都只能维持一小段时间，看到别人工作很激情，我也是羡慕的，可是回到自己身上，还是使不上劲。就像现在的学习编程，其实我要学的东西还有很多很多，我能做的也有很多很多，就是去学，可是我也会产生无力感，这已经是我最不抵触的事情了。当然，可能我最想做的事情就是躺在床上玩手机，就这么躺着，然后等良心过意不去了，爬起来学一会儿。

学习真的是急不来的。我很焦虑，想要快速学会技能好找到工作。可是，我不是天才，我每天学习的，也会忘记，需要再复习，还有许多未知的，等我去学习，静下心来，才能学得进去。

从小的学习，就好像是有什么已经在前面铺路，都是安排好的，照着做就好了，不用思考，不用计划。现在突然把这份权力赋予了自己，反而有些不知所措。不要求计划的多么完美缜密，但是心中有路了，总是好上道点。

链表中环的检测，也可以利用快慢指针，因为倘若链表中存在环，那么这个遍历会一直持续下去，那么快慢指针终有相遇的时候。代码如下：

第三个是删除链表倒数第N个结点。这个一开始即使看解法，我也没有看懂（真的愚蠢）。其实从一开始我就没看清题目的意思，题中列表和N值是给定的。使用快指针明确走到第N个结点，然后慢指针开始走，当快指针走到链尾，慢指针所在的位置就是倒数第N个。当然遍历两边，直接找到倒数N的位置也是可以的

最后要讲一下合并两个有序列表，首先，是看有没有任一个表是空列表，那合并后就是非空那一列表。若两个链表都是非空链表，就分别拿出链表的第一个结点开始比大小，小的那个作为第一个结点，再把较大的那个与小的那个结点的下一个结点作比较，运用递归的方式，始终保持较小值的结点在前面

链表的五种操作方式终于写完了，其实除了快慢指针，还有其他方式也可以解题，这里着重介绍了快慢指针的用法。关于python的数据结构和算法，我觉得我对python面向对象这个特性以及用法什么的还很不清楚，然后我找到了两个不错的资源，希望能尽早看完，这里也贴一下。

      python抓基金公告
    

  

      

    
    

      
        
这是我第一个写出来有实际应用功能的程序，感觉很开心。这个程序实现了一个很简单的功能，就是抓取证券报纸上的板块信息，把基金公告筛选出来，把公告序号、标题以及链接写入excel中。


   这里我要理一下我写的思路，因为之后的程序我是在这个基础上不断借鉴和改进的。最开始是看了《Python网络爬虫从入门到实践》（简称实践书）前几章，跟着作者爬了豆瓣电影网站，简单的把电影名字爬了下来。我就在想可不可以把评分和打星评级也爬下来，然后就在我哥的帮助下实现了这个功能。其中认识到了pdb这个模块，用于断点测试，可以不断试错，很好用。  


然后因为我自己的日常工作，有这么一个过程，要获得基金公告这个数据源，然后把数据录入数据平台中。之前查询基金公告一直是通过人工登录到四大证券报纸上，按照”基金“关键词把公告的序号和公告标题输入excel中。鉴于证券日报不需要账户密码，并且公告披露页面公告齐全，我就选择了证券日报作为第一个”小目标“。一开始写的时候，我就是想到哪写到哪，只有一个很简单的雏形，在我哥的提点下，一开始的难点顺利解决，接下来就是不断搜索方法，走一步看一步的把我想要实现的功能给实现。


  因为是第一遍看实践书，其实掌握的不是很ok，爬虫的几种抓取方式我也是一知半解，就跟着用了requests模块，利用chrome浏览器的审查元素，获取了headers里一些信息，给出指定的url，就获取到了网页源代码。接下来是解析代码，看书里方法也有很多，beautifulsoup，lxml,正则表达式。一开始我也挺混乱的，哪个好用就用哪个，也不能静下心来看每个方法的官方文档介绍（当然现在也没去看），然后就疯狂使用正则表达式的re.sub,bs4的soup.find这几个方法反复使用，期间还慢慢了解了字符串的一些切片和去空格等用法（弱爆啦~~）。里面最值得一提的思想就是，我先把公告的序号和链接抓取到了，放入一个字典里面e中。然后又搞了一个字典c，放的是公告的序号，和序号对应下的公告集合，用列表装着。但是最终我要输出序号和对应的公告标题，而字典的key值又必须是唯一的，这时候我想到了在leetcode上看到的最优解法里面，可以把字典反转，把值作为键，因为值是公告标题是不重复的，即使重复了也只是序号被改变，没有序号作为键影响来的大。所以下一步我就遍历了字典c每个键下面的值，值是列表，再次循环，只要标题中含有基金，就把这个标题作为键，把对应的序号作为值，放入字典d中。最后写入到excel中就用到了d和e两个字典。


  再接下来就是写带有账户密码的中证报了，同样具有账号密码，中证报用了网上一个引用cookie的方法，就解决了（那篇借鉴的文章忘记收藏了，以后最好养成随手收藏的习惯，不然回去找找不到也是蛮心累的）。但是！！上证报，不按套路出牌，竟然不能直接用cookie加上requests解决，把我折磨了好久，到现在还没想出解法，我逐步猜测，那应该是动态网页，会自动加载的，用了AXAJ技术，需要用到seleium(说实话第一遍看实践书我肯定是没看懂，不然遇到这种情况应该马上翻书了而不是脑袋放空了半小时。。。)因为上证报没有解决，我只能先去处理证券时报，好在证券时报也比较好，很容易就抓取下来想要的数据，因为是第三份报纸的爬取，在前两份报纸的经验上，我代码略微进行了优化（有进步总是好的=.=），感觉beautifulsoup用的也稍微熟悉了些，抽空一定要把文档给读懂的读完。接下来的计划和目标就是专攻上证报了，毛估估需要用到selenium和xpath模块，而这两个我也不会，只能寄托于广大好心的网友，在线学习她们的博客。


  （基础、算法、实践）路漫漫。。。

      反转链表（leetcode 206)
    

  

      

    
    

      
        
反转一个单链表。


示例:


输入: 1->2->3->4->5->NULL
输出: 5->4->3->2->1->NULL


解法：


一开始看到这个解法我是懵逼的，然后看解法下面的评论，开始了解这个表达方式，其实包含了一种我不知道的意思。就是while循环中的模块，是一个元祖形式，可以写为（rev,rev.next,p)=(p,rev,p.next)（这里可以看出，有看不懂的表达式是很正常的，这时候就要多搜索，从而获得更多知识点，逗号放在print中，可以不换行，具体的用法可以参考python多元赋值 https://blog.csdn.net/qq277511467/article/details/50915739


之后的理解相对就好些了。看到回复中有个便于理解的回复，下面贴过来：
前置条件：迭代指针：p = head、结果指针：rev = none
以1->2->3->4->5为例：
过程：
rev:None
第一层循环
rev:1->2->3->4->5    rev = p
rev:1->None    rev.next = rev
p:2->3->4->5    p = p.next
第二层循环
rev:2->3->4->5    rev = p
rev:2->1->None    rev.next = rev
p:3->4->5    p = p.next
第三层循环
rev:3->4->5    rev = p
rev:3->2->1->None    rev.next = rev
p:4->5    p = p.next
第四层循环
rev:4->5    res = p
rev:4->3->2->1->None    rev.next = rev
p:5    p = p.next
第五层循环
rev:5    rev = p
rev:5->4->3->2->1->None    rev.next = rev
p:None    p = p.next


end…

      习题2.4 递增的整数序列链表的插入
    

  

      

    
    

      
        
本题要求实现一个函数，在递增的整数序列链表（带头结点）中插入一个新整数，并保持该序列的有序性。
函数接口定义：
List Insert( List L, ElementType X );
其中List结构定义如下：
typedef struct Node PtrToNode;
struct Node {
    ElementType Data; / 存储结点数据 /
    PtrToNode   Next; / 指向下一个结点的指针 /
};
typedef PtrToNode List; / 定义单链表类型 */
L是给定的带头结点的单链表，其结点存储的数据是递增有序的；函数Insert要将X插入L，并保持该序列的有序性，返回插入后的链表头指针。
裁判测试程序样例：


输入样例：
5
1 2 4 5 6
3
输出样例：
1 2 3 4 5 6


打铁要趁热，这两道题其实存在相似之处，都利用了链表的插入，即在找到适当的位置后进行插入操作
