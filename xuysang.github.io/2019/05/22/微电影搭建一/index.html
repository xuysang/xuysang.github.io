微电影搭建一

为了避免到了睡觉的点还写不完，提早开工，从中午就开始写啦。这个微电影是我在b站上，看到一个up主上传的视频，正好不久前看了Flask Web开发这本书，虽然作者写的很细致了，但是没有项目跟着练习，总觉得学了也还是没有底。其实这个视频我目前大概看到一半了，但是最近开始写博客了，总觉得有必要自己再写一写，这样理解的才更深而且以后还可以来回顾复习。说实话，如果不写这个博客，我已经忘了前面都讲了些啥（囧）。于是乎，我就一边开着二倍速从头回顾，一边开始写。
最开始老师介绍了微电影网站最后的成品，是达到什么样的效果。然后从架构上，一个微电影网站分为前台和后台两部分。

首先需要创建虚拟环境，然后激活虚拟环境。（page6）

再接下来就是蓝图（blueprint)：一个应用中或跨应用制作应用组件和支持通用的模式。

大概是对抽象概念不够敏感，看书的时候还是不太能理解，结合项目来，蓝图的作用就是将不同功能模块化(例如前台后台)，优化项目的结构，增强可读性，易于维护。分别给前台后台构建蓝图后，就可以在views调用各自的蓝图。

再再接下来就是数据模型的设计（p48)

里面用了一个Flask-SQLAlchemy的扩展，SQLAlchemy是一个关系型数据库框架，既提供高层ORM，也提供数据库原生SQL的低层功能。

具体步骤：
1.需要在虚拟环境中下载Flask-SQLAlchemy，然后在models里面开始定义mysql数据库连接，然后定义模型。
2.先在本地的mysql中创建一个数据库，然后回到编辑器，定义数据库连接。这里一定要插一句的是，因为python3版本，’mysql+pymysql’这样才能连接到本地的数据库

3.定义会员模型（p48)，每个模型要定义主键，一般是id，repr()是返回一个具有可读性的字符串表示模型，供调试和测试使用。
